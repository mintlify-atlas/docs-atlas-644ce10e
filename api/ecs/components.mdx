## Overview

Components are atomic bundles of data attached to entities. Each component has a `type` field and optional `data` field containing networked state.

## Core Types

### Component

```typescript
type Component<
  type extends string = string,
  D extends NetworkedComponentData | undefined = undefined
> = {
  type: type
  active?: boolean
} & (D extends undefined ? {} : { data: D })
```

The base component type. All components extend this structure.

<ParamField path="type" type="string" required>
  Unique identifier for the component type
</ParamField>

<ParamField path="active" type="boolean">
  Whether the component is currently active
</ParamField>

<ParamField path="data" type="NetworkedComponentData">
  Networked data object containing the component's state
</ParamField>

### NetworkedComponentData

```typescript
type NetworkedComponentData = Record<string,
  null | boolean | string | number |
  string[] | number[] |
  (string[] | undefined)[] | XY | Date
>
```

Valid types for component data that can be synchronized over the network.

## Built-in Components

<AccordionGroup>
  <Accordion title="Position" icon="location-dot">
    The entity's position and movement in world space.

    ```typescript
    Position(props?: PositionProps): Position
    ```

    **Type Definition**
    ```typescript
    type Position = Component<"position", {
      x: number
      y: number
      z: number
      aim: XY
      facing: -1 | 1
      follows: string | null
      friction: boolean
      flying: boolean
      gravity: number
      heading: XY
      offset: XY
      pointing: Oct
      pointingDelta: XY
      recoil: number
      rotating: number
      rotation: number
      speed: number
      standing: boolean
      stop: number
      swimming: boolean
      tether: null | XYZ & { dist: number }
      velocity: XYZ
      velocityResets: number
    }>
    ```

    **Methods**
    - `setPosition({ x?, y?, z? })` - Update entity position
    - `setVelocity({ x?, y?, z? })` - Set velocity vector
    - `impulse({ x?, y?, z? })` - Add impulse to velocity
    - `setHeading(xy: XY)` - Set target heading
    - `clearHeading()` - Clear movement heading
    - `interpolate(world, delta)` - Get interpolated position
    - `xyz()` - Get position as XYZ object

    **Example**
    ```typescript core/src/ecs/entities/characters/Skelly.ts
    position: Position({
      x: 0,
      y: 150,
      z: 50,
      velocityResets: 1,
      speed: 125,
      gravity: 0.3
    })
    ```
  </Accordion>


    **Type Definition**
    ```typescript
    type Health = Component<"health", {
    ```

    - `getKDA()` - Get kills/deaths/assists object
    - `setKDA(kda)` - Update KDA stats

    **Example**


## Serialization

### serializeComponent

```typescript
serializeComponent(c: Component): NetworkedComponentData
```

Serialize a component's data for network transmission.

### deserializeComponent

```typescript
deserializeComponent(
  c: Component,
  data: NetworkedComponentData
): void
```

Deserialize network data into a component.
